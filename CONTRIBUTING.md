# 기여 가이드

> 본 패키지는 한국 송무 변호사 사용을 가정합니다.
> 기여 환영하지만, 다음 원칙을 지켜주세요.

---

## 절대 커밋하지 말아야 할 것

### 의뢰인 정보·개인정보 일체

- 실제 사건번호 (`YYYY가합NNNN` 등)
- 의뢰인·상대방 성명
- 주민등록번호 (`YYMMDD-NNNNNNN`)
- 휴대전화번호
- 주소
- 사무소 내부 절차의 구체적 사례

**테스트·예시는 모두 가상 사건으로 작성하세요.** 본 패키지의 강의용 가상사건(`김민철 vs 이정숙` 등)을 참고할 수 있습니다.

### 사용자별 비밀

- API 키 (Gemini, OpenAI, Anthropic 등)
- 비밀번호·토큰
- `secrets.env`, `.env` 파일

`.gitignore`가 차단하지만, 실수 방지를 위해 PR 전 `git diff` 직접 확인 권장.

### 사무소 인스턴스 파일

- `CLAUDE.md` (각 플러그인의 사용자 인스턴스)
- `~/사건/_사건정보관리표.csv`

→ 모두 `.gitignore`됨. 템플릿(`CLAUDE.md.example`)만 커밋.

## 환영하는 기여

- 새 스킬·가이드 추가 (특히 분야별 워크플로우)
- 자동 설치 스크립트의 OS 호환성 개선
- 데이터 보호 Hook 패턴 추가
- 문서·번역 개선 (영어·일본어 등)
- 버그 리포트·재현 방법

- [ ] API 키 누출 없음
- [ ] 새 의존성 추가 시 README 갱신
- [ ] 데이터 보호 Hook과 충돌 없음
- [ ] Mac + Linux 두 환경에서 테스트 (가능하면)
