# mock-hearing(모의변론) 스킬 적용 결과

적용일: 2026-05-30
참고: [crealwork/yc-office-hours](https://github.com/crealwork/yc-office-hours)

## 무엇을 했나

yc-office-hours 스킬을 참고하여 JuriSupport 플러그인에 **모의변론(mock-hearing) 스킬**을 신규 추가했다. 사용자가 선택한 네 가지 발전 방향을 하나의 스킬로 통합했다:

1. 모의변론 신규 스킬
2. 반아첨 어조 규칙 도입
3. 스킬 다중파일화
4. 구조화 평결 프레임

핵심 아이디어: yc-office-hours가 "YC 파트너가 사업 전제를 공격"하는 구조라면, 그 법률 대응물은 **상대방 대리인·재판부 관점에서 우리 서면·사건이론의 약점을 제출 전에 공격하는 모의변론**이다.

- `brief-protocol` = 서면을 **쓰는** 스킬
- `mock-hearing` = 그 산출물을 **두들겨 보는** 스킬 (정본 등록·외부 발송 안 함)

## 신규 파일 (4개, 다중파일 구조)

위치: `plugins/jurisupport/skills/mock-hearing/`

| 파일 | 역할 | yc-office-hours 대응 |
|---|---|---|
| `SKILL.md` | 워크플로우, 3개 모드, 순차 게이트, brief-protocol 연계 | SKILL.md |
| `question-bank.md` | 모드·단계별 압박 질문 은행 | question-bank.md |
| `evaluation-rubric.md` | 5개 강도 지표 채점표, 4단 평결, 15종 패소 함정 | evaluation-rubric.md |
| `adversary-voice.md` | 반아첨 어조, 금지 표현, 치환 규칙 | partner-voice.md |

## 핵심 설계

### 워크플로우 (순차 게이트)
오프닝 → 쟁점 사다리(표층→요건→증거→법리→입증책임) → 심화 공격(모드별) → 핵심전제 반박 → 평결 → 보강 과제

### 3개 모드
| 모드 | 대상 | yc 대응 |
|---|---|---|
| THEORY (사건이론) | 청구원인/항변 구성, 착수 단계 | IDEA |
| BRIEF (서면) | 작성된 초안의 문장 단위 약점 | CAMPAIGN |
| HEARING (변론대비) | 재판부 석명·상대 구두주장 예측 | GROWTH |

### 4단 평결
| 평결 | 의미 | yc 대응 |
|---|---|---|
| 제출 가능 (PROCEED) | 주요 쟁점 우위 + 약점 통제됨 | PROCEED |
| 보강 후 제출 (REINFORCE) | 핵심 약점 있으나 보강 가능 | TEST FIRST |
| 전략 재구성 (REFRAME) | 현 사건이론 승산 낮음 | PIVOT |
| 출구 검토 (EXIT) | 본안 승산 희박, 화해·조정·취하 | KILL |

### 강도 지표 5개 (각 1~5점)
요건사실 충족도 / 증거 뒷받침 / 법리 정합성 / 입증책임 위치 / 반박 견딤성

### 패소 함정 탐지 (15종)
입증책임 오해, 요건사실 누락, 시효·제척기간 간과, 당사자적격 흠결, 기판력·중복제소, 판결 오인용, 직접인용 불일치, 변론주의 위반, 청구취지-원인 불일치, 의제자백 리스크, 항변 미대비, 감정 주장 의존, 과잉 주장, 단일 증거 의존, 증거수집 경로 오판
(yc-office-hours의 tarpit detection 대응)

### 자료 검증·역공 스택 (2026-05-30 추가)
모의변론이 자료를 **두 방향**으로 쓴다:
- **방어(verify)**: 우리 인용 법령·판결·교과서가 실존하고 원문과 일치하는지 확인
- **역공(attack)**: 상대가 들고 나올 반대 판결·교과서 반대 견해를 같은 도구로 먼저 찾아 제시

| 자료 | 도구 | 부재 시 |
|---|---|---|
| 교과서 DB | `legal-books` (localhost:8766) | 책 0권/서버 미응답 → 자동 스킵 + 안내 |
| 법령 | `korean-law` MCP | 필수, 스킵 안 함 |
| 판례 1차 | `korean-law` MCP | 1차 |
| 판례 2차 | `beopgoeul-search` (법고을 무료) | toolkit 없으면 자동 스킵 |
| 판례 보조 | lbox.kr | 계정 보유 시 검색 결과 반영 |

역공 자료가 발견되면 법리 정합성·반박 견딤성 점수를 낮추고 "상대가 이걸 든다"로 명시. 각 모드(THEORY/BRIEF/HEARING) Phase 2에서 이 스택을 양방향으로 돌린다.

### 반아첨 어조
금지 표현: "탄탄합니다 / 문제없어 보입니다 / 충분합니다 / 잘 작성되었습니다 / 무난합니다 / 승소 가능성이 높습니다(근거 없이)"
→ 모든 지적은 특정 문장·쟁점·증거를 지목하고, 상대가 어떻게 반박할지까지 명시.
(검증 맥락에서만 적용. 서면 본문·의뢰인 보고서에는 적용 안 함.)

## 사무소 정책 준수 확인

- 인용 검증: korean-law(1차) + 법고을(2차)
- 서면 문구 제안 시 "판례" 대신 "판결"(또는 "판단") 사용 명시
- 증거수집은 사실조회보다 공적 기관 문서제출명령 우선
- em-dash(긴 대시) 미사용: 신규 4개 파일 0건 (작성 중 2건 발견 후 제거)
- 의뢰인 보고서 어조·서면 격식체와 분리 (검증 전용 어조)

## 변경하지 않은 것

- `CLAUDE.md` / `CLAUDE.md.example`: 어조 규칙은 검증 맥락 전용이라 플레이북 전역 톤에 넣지 않음 (서면은 격식체 유지)
- `plugin.json`: 스킬은 `skills/` 디렉토리에서 자동 발견되므로 별도 스킬 등록은 불필요. 배포 버전은 `0.2.3`으로 올림.
- `brief-protocol` 자체의 다중파일 분리: 위험·대규모라 제외. 신규 스킬을 다중파일 모범 사례로 삼고 cross-ref만 연결

## 동작 확인 방법

1. `/jurisupport:mock-hearing` 호출 → 모드 선택 프롬프트
2. 준비서면 초안으로 BRIEF 모드 실행 → 쟁점사다리·문장단위 적발·채점표·4단 평결이 순차 게이트로 진행되는지
3. `/jurisupport:brief-protocol` 실행 시 Phase 3 통과 후 모의변론 권유가 나타나는지
4. 검증 출력에 금지 표현("탄탄합니다" 등)이 안 나오는지

## 후속 정리

`plugins/jurisupport/README.md`의 스킬 목록 구분자는 em-dash 대신 ASCII 하이픈으로 정리했다. 서면·플러그인 문서에서 em-dash 사용을 줄이는 사무소 선호와 맞춘다.
