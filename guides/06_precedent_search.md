# 판례 검색 통합 워크플로우 — 법고을 자동 검색 → lbox 수동 폴백

> 본 패키지의 판례 검색 정책: **무료 공식 사이트(법고을)를 1차 자동 검색하고, 부족할 때 유료(lbox)는 사용자가 수동으로**.

---

## 정책 요약

| 사이트 | 비용 | 자동화 정책 | 1차 vs 폴백 |
|---|---|---|---|
| **법고을** (lx.scourt.go.kr) | 무료, 비로그인 | **Selenium 자동 검색 허용** (정부 공식, 약관상 자동화 명시 금지 없음, 양심적 사용) | **1차 자동** |
| **lbox** (lbox.kr) | 유료 | LLM 자동 접근 절대 금지 (이용약관 위반) | **2차 폴백 — 사용자 직접 검색** |

이 정책은 본 패키지의 데이터 보호 Hook이 강제: `lbox.kr` 도메인에 대한 WebFetch/WebSearch는 자동 차단됩니다.

법고을 자동 검색을 사용하려면 toolkit 설치 필요:

```bash
bash ~/jurisupport-plugins/toolkit/beopgoeul/install.sh
```

설치 후 `beopgoeul-search` 스킬이 활성화되어 클로드가 직접 검색 가능.

---

## 표준 워크플로우 (5단계)

### Step 1. 사용자 → 클로드: 쟁점 + 검색 요청

```
이 사건 시효 쟁점에 대한 판례 찾아줘.
```

### Step 2. 클로드: 법고을 자동 검색 (toolkit 설치 시)

`beopgoeul-search` 스킬 활성화된 경우 클로드가 자동으로:

```bash
~/jurisupport-beopgoeul/scripts/search.sh "소멸시효 채무승인" --max 5 --format json
```

→ 결과 표로 정리하여 사건번호·법원·선고일·PDF URL·요약 즉시 표시.

#### Toolkit 미설치 시 (fallback)

클로드가 검색 키워드와 법고을 검색 URL만 안내:

> 다음 URL을 브라우저에서 클릭해주세요:
> https://lx.scourt.go.kr/search/total/get/list?search_txt=소멸시효+채무승인&searchCategory=prec

### Step 3. 사용자: 브라우저에서 법고을 직접 검색 + PDF 다운로드

브라우저로 URL 클릭 → 결과 목록 확인 → 사건 선택 → PDF 다운로드 → 사건폴더에 저장.

### Step 4. 사용자 → 클로드: 결과 분석 요청

```
06_참고판례/에 PDF 3건 저장했어. 본 사건 쟁점에 적용해서 정리해줘.
```

클로드가 kordoc 또는 내장 PDF 리더로 본문 읽고 분석.

### Step 5. (필요 시) lbox 폴백

법고을에서 적합한 판결을 못 찾았을 때:

```
법고을에 없어. lbox에서 어떤 키워드로 찾을지 알려줘.
```

클로드 응답 예시:

> 법고을에 적합한 판결이 없으셨다면 lbox.kr에서 다음을 시도해 보세요.
>
> **lbox 검색 키워드 (자동 검색은 약관 위반이므로 사용자가 직접 검색해야 합니다)**
> 1. `소멸시효 채무승인` + 하급심 포함 필터
> 2. `시효이익 포기 묵시적` (lbox는 자연어 검색 정확도가 더 높음)
> 3. 사건번호 `2020다234567` 등 직접 입력
>
> lbox.kr 로그인 → 검색 → PDF 다운로드 → `06_참고판례/`에 저장 → 알려주세요.

⚠️ 위 단계에서 **클로드는 절대 lbox.kr에 WebFetch·WebSearch·Bash로 접근하지 않습니다.** 현재 데이터 보호 Hook은 WebFetch/WebSearch의 lbox.kr 접근을 차단하며, Bash/curl/playwright 접근은 스킬 정책상 금지합니다.

---

## 사용자가 클로드에게 시키면 안 되는 것

| 요청 | 클로드 반응 |
|---|---|
| "lbox에서 찾아서 가져와줘" | 거절 + 키워드 안내 + 사용자 직접 검색 안내 |
| "이 lbox URL 가져와줘" | Hook이 차단 |
| "lbox 자동 로그인해서 검색해줘" | 거절. Bash·playwright 자동화도 스킬 정책상 금지 |

## 결과물 저장 구조

```
사건명_사건번호/
└── 06_참고판례/
    ├── 법고을/
    │   ├── 대법원_2013다64793.pdf
    │   └── 대법원_2018다234567.pdf
    ├── lbox/
    │   └── 서울고법_2020나123456.pdf
    └── _검색이력_2026-05-18.md   ← 어떤 키워드·어떤 사이트에서 찾았는지
```

`_검색이력_*.md`에는 검색에 사용한 키워드 + 결과 사건번호 + 출처(법고을/lbox)를 메모. 다음에 같은 쟁점 다시 검색할 때 빠르게 재활용.

---

## 직접인용 시 주의 (모든 사이트 공통)

- 사건번호는 PDF 안 원문과 글자 단위 일치 확인
- 직접인용(`"..."`)은 PDF 원문과 글자 단위 일치
- 일치 못 시키면 따옴표 제거하고 간접인용
- 본문 표현은 "**판결**" 사용 (`판례` 금지)
- 하급심 판결 인용 시 항소·상고 결과 별도 확인
