# 사건정보 관리표 입력 가이드

> JuriSupport 같은 전용 사건관리 시스템을 쓰지 않는 변호사를 위한 엑셀/CSV 기반 사건관리 방법.
> 본 가이드대로 입력해 두면 클로드코드가 "지난주 기일 정리해줘" 같은 질문에 정확히 답할 수 있습니다.

---

## 1. 파일 위치

표준 위치 (권장):

```
~/사건/
├── _사건정보관리표.csv         ← 마스터 표 (이 파일)
├── _사건정보관리표.xlsx        ← 엑셀 작업용 사본
├── SAMPLE-001_홍길동_대여금/
├── 2026가단11111_김아무개_손해배상/
└── 2025가합22222_박아무개_이혼/
```

- 사건폴더 이름 규칙: `{사건번호 또는 사건ID}_{의뢰인성명}_{사건명요약}`
- CSV가 진본 (Single source of truth), 엑셀은 작업·편집 용도

## 2. 컬럼 정의

| 컬럼 | 의미 | 예시 | 필수 |
|---|---|---|---|
| 사건ID | 사무소 내부 고유번호 (수임번호) | `2026-001` | ✅ |
| 사건번호 | 법원 사건번호 | `2026가단12345` | (소제기 후) |
| 사건명 | 사건 종류 + 의뢰인 호칭 | `홍길동 대여금 청구` | ✅ |
| 법원 | 관할 법원 | `서울중앙지방법원` | (소제기 후) |
| 재판부 | 배당 재판부 | `제20민사부` | (배당 후) |
| 사건단계 | 1심 / 항소심 / 상고심 / 보전 / 집행 등 | `1심` | ✅ |
| 청구금액 | 청구취지상 금액 (원 단위, 정수) | `80000000` | (해당 시) |
| 의뢰인유형 | `원고` / `피고` / `청구인` / `상대방` 등 | `원고` | ✅ |
| 의뢰인성명 | 개인 또는 법인명 | `홍길동` | ✅ |
| 의뢰인연락처 | 휴대전화 | `010-1234-5678` | ✅ |
| 상대방성명 | 상대방 (다수면 첫 사람) | `김철수` | ✅ |
| 상대방연락처 | 알 수 있으면 | `010-9876-5432` | |
| 소제기일 | YYYY-MM-DD | `2026-05-16` | (소제기 후) |
| 첫기일 | 변론(준비)기일 1회 | `2026-08-20` | |
| 다음기일 | 다음 예정 기일 | `2026-11-17` | |
| 수임일 | 위임계약일 | `2026-05-10` | ✅ |
| 수임료 | 보수액 (원) | `3300000` | ✅ |
| 주된쟁점 | 키워드 (콤마 구분) | `소멸시효, 채무승인` | ✅ |
| 비고 | 기타 메모 | `의뢰인 해외출장 잦음` | |
| 상태 | `진행중` / `종결` / `중지` 등 | `진행중` | ✅ |

## 3. 입력 방법 두 가지

### A. 엑셀에서 직접 입력 (권장)

1. `_사건정보관리표.xlsx`를 열고 행 추가
2. 빈 셀은 그대로 두기 (필수 컬럼만 채워도 OK)
3. 저장 후 별도로 CSV로 내보내기 (`파일 → 다른 이름으로 저장 → CSV UTF-8`)
4. 또는 처음부터 CSV만 편집 (수기 또는 텍스트 에디터)

### B. 클로드코드에게 단계별 질문받으며 입력

```
새 사건 등록할게. 사건정보관리표에 한 행 추가해줘.
하나씩 질문해줘.
```

클로드코드가 다음과 같이 진행:
- "사건ID는 무엇인가요? (예: 2026-005)"
- "의뢰인 성명?"
- "사건 종류?" ...
- 모두 입력 후 CSV에 한 행 자동 추가

## 4. 클로드코드 활용 명령어 예시

사건정보관리표 + 사건폴더 구조를 갖춰두면 다음 질문이 가능:

| 질문 | 클로드 동작 |
|---|---|
| `2주 이내 기일 있는 사건 정리해줘` | CSV의 `다음기일` 컬럼 정렬 |
| `이번 달 신규 수임 건 정리` | `수임일` 필터 |
| `홍길동 의뢰인의 진행중 사건 모두 보여줘` | `의뢰인성명` 매칭 |
| `소멸시효 쟁점 들어간 사건들 보여줘` | `주된쟁점` 검색 |
| `2026-005 사건폴더 열고 사실관계 요약` | CSV → 사건폴더 매칭 → 폴더 읽기 |
| `종결되지 않은 사건 중 6개월 이상 다음기일 비어있는 것` | 두 컬럼 결합 검색 |

## 5. 사건폴더 표준 구조 (권장)

```
2026-001_홍길동_대여금/
├── 01_위임계약/
│   ├── 위임계약서.pdf
│   └── 소송위임장.pdf
├── 02_의뢰인자료/
│   ├── 차용증_원본.pdf
│   └── 송금확인증.pdf
├── 03_소송서류/
│   ├── 001_소장.pdf
│   ├── 010_답변서.pdf
│   ├── 020_준비서면.pdf
│   └── 030_변론조서.pdf
├── 04_갑호증/
├── 05_을호증/
├── 06_참고판례/
├── 07_의뢰인소통/
│   ├── 카톡_2026-05-10.md
│   └── 통화메모_2026-05-15.md
└── _사건메모.md
```

`_사건메모.md`에는 의뢰인 호칭·민감 정보·전략 등 자유롭게 메모.

## 6. JuriSupport와의 비교

| 항목 | CSV 관리 (본 가이드) | JuriSupport |
|---|---|---|
| 비용 | 무료 | 월 구독 |
| 진입장벽 | 낮음 (엑셀만 알면 됨) | 가입·설정 필요 |
| 사건 메타정보 | CSV 한 줄 | DB 통합 관리 |
| 문서 버전관리 | 수동 | 자동 |
| 기일·할일 알림 | 수동 (Calendar 별도) | 자동 |
| MCP 통합 | 없음 (파일 읽기만) | 풍부 (조회·등록·수정 모두) |
| 다중 사용자 협업 | 어려움 | 가능 |
| 권장 대상 | 1인 사무소, 시작 단계 | 다인 사무소, 본격 운영 |

→ 본 CSV 방식으로 시작하고, 사건수·인원이 늘면 JuriSupport로 전환하는 경로 추천.

## 7. 데이터 보호

본 CSV에는 의뢰인 성명·연락처·사건번호 등 **민감 정보**가 다수 포함됩니다.

- ❌ Google Drive·Dropbox 공유링크로 만들지 말 것
- ❌ 외부 번역기·요약 서비스에 업로드하지 말 것
- ✅ 로컬 디스크 + 개인 클라우드 (비공개) 보관
- ✅ 외장 SSD 정기 백업
- ✅ 본 패키지의 **데이터 보호 Hook** 활성화 ([guides/00_security.md](00_security.md))
